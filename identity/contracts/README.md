# contracts · Verifier Contract (Soroban)

Soroban contract that **verifies the ZK proof** (Groth16) and **registers** the address as
KYC-verified, guaranteeing *one person = one registration* via `nullifier`.

> 📐 Design in the vault: `Contrato Verificador (Soroban)`, `Flujo de KYC` (Phase 3),
> `Modelo de Datos`. Starting point: `groth16_verifier` from `soroban-examples`
> (see `09 - Tools/Verificadores ZK de referencia`).

## ⚠️ Curve: BLS12-381 (not BN254)

The **official** Groth16 verifier from soroban-examples uses
**BLS12-381** host functions (`crypto::bls12_381`, CAP-0059, available). Native BN254 / Poseidon
primitives remain **CAP-0074/0075 (proposals, not available in SDK)**. Therefore:

- Circuit compiles with `--prime bls12381` (see `identity/circuits`).
- This contract verifies pairing on BLS12-381, with the **same logic** as the
  official `groth16_verifier`, plus: issuer root, address binding, and nullifier.
- SDK pinned to `soroban-sdk = 25.x` (includes `crypto::bls12_381`).

## Interface

| Function | What it does |
|---|---|
| `init(admin, trusted_root, vk)` | Stores trusted issuer root and verifying key. Once only. |
| `verify_and_register(address, proof, public_inputs)` | `require_auth` + issuer root + address binding + nullifier unused + Groth16 pairing, then registers. |
| `verify_proof(proof, public_inputs) -> bool` | Pure pairing verification (no registration). |
| `is_verified(address) -> bool` | Query used by Layer 2 dApps. |
| `address_field_hash(address) -> BytesN<32>` | Expected `addressHash` for the address (used by off-chain prover). |

**Public inputs** (order agreed with circuit), each `BytesN<32>` big-endian:

```
[ commitment, nullifier, issuerRoot, addressHash ]
```

### Address binding

`addressHash = sha256(address.to_xdr())` with top 2 bits set to 0 (so value
is `< r_bls12381` and there is no modular reduction: on-chain and circuit representations
are byte-identical). Off-chain prover must use exactly `address_field_hash`.

## Build & test

```bash
stellar contract build          # compiles to wasm (target wasm32v1-none)
cargo test -p kyc_verifier      # contract tests
```

Tests use real artifacts from the ZK pipeline (`src/testdata.rs`) to verify that
proof generated by Circom+snarkjs **verifies with `groth16_verifier` logic**, plus
negative paths (untrusted issuer, address mismatch, nullifier replay).

### Regenerate `testdata.rs` (if circuit changes)

```bash
# 1) addressHash of fixed test address
cargo test -p kyc_verifier print_address_hash -- --ignored --nocapture   # -> ADDRESS_HASH_HEX=...
# 2) input + proof + test data (see identity/circuits/README)
cd ../circuits
node scripts/gen_input.mjs <addressHashDecimal>
bash scripts/prove.sh
node scripts/gen_testdata.mjs
```
